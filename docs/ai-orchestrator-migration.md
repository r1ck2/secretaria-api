## Overview

The AI Orchestrator is deployed alongside the existing FlowEngine. A feature toggle (`use_ai_orchestrator`) controls which engine processes incoming messages. Migration is zero-downtime and fully reversible.

---

## Phase 1 â€” Deploy (no traffic change)

1. Deploy the new code (includes all AI Orchestrator components)
2. Run the migration to create the toggle setting:
   ```bash
   npx sequelize-cli db:migrate
   ```
3. Verify the setting exists with `value = 'false'`:
   ```sql
   SELECT * FROM cad_settings WHERE key = 'use_ai_orchestrator';
   ```
4. Confirm the FlowEngine still handles all traffic (toggle is off)

---

## Phase 2 â€” Activate for testing

1. Ensure at least one agent has a valid `openai_api_key` in `cad_agents`
2. Enable the toggle via API:
   ```bash
   curl -X PATCH https://your-api/api/v1/settings/ai-orchestrator \
     -H "Authorization: Bearer <admin-token>" \
     -H "Content-Type: application/json" \
     -d '{"enabled": true}'
   ```
3. Send a test WhatsApp message and verify:
   - Response is generated by OpenAI (not mock)
   - Log entry with `module = 'ai_orchestrator'` appears in `cad_logs`
   - Session created in `mv_flow_sessions`

## Rollback Procedure

Disable the toggle at any time â€” traffic immediately reverts to FlowEngine:

```bash
curl -X PATCH https://your-api/api/v1/settings/ai-orchestrator \
  -H "Authorization: Bearer <admin-token>" \
  -H "Content-Type: application/json" \
  -d '{"enabled": false}'
```

Or directly in the database:
```sql
UPDATE cad_settings SET value = 'false'
WHERE key = 'use_ai_orchestrator' AND is_admin = true;
```

No code deployment required for rollback.

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| No AI response | Agent has no `openai_api_key` | Add key to `cad_agents` |
| Tool calls fail silently | Calendar not connected | Connect Google Calendar for the professional |
| Fallback message sent | OpenAI API error | Check `cad_logs` for `action = 'openai_error'` |
| Messages not sent | WhatsApp disconnected | Reconnect instance in WhatsApp settings |
| Toggle has no effect | Old code still running | Restart the API server |
